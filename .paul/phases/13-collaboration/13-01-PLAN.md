<objective>
## Goal

Add collaboration features: team members can comment on tasks, clients can request revisions via flagged comments in the portal — unified through a single comment system with a `revision_requested` type flag.

## Purpose

Currently there's no way for team members to discuss context on a task, and no structured way for clients to request revisions. Both needs are served by comments: team members leave general comments, clients leave revision requests (which appear as flagged comments for the team).

## Output

- New `is_revision` boolean column on `comments` table
- Comment UI in admin task detail dialog (view only)
- Comment form in team task detail form (post + view)
- Comment form in client portal task detail (flagged as revision requests)
- Server action for team to post comments
- Server action for clients to post revision-requested comments
</objective>
