<objective>
## Goal
Fix photo/design file upload failures across all admin surfaces by adding auth verification to the upload API route, improving error handling, and ensuring consistent client-side validation.

## Purpose
Uploads are currently failing silently with generic "Upload failed" messages. The upload API route has no auth guard (deferred security issue from Phase 15), and client components discard server error details. This plan resolves the functional failure and closes the security gap.

## Output
- Upload API route with admin auth guard and detailed error logging
- Client upload components with improved error surfacing and consistent validation
- Verified upload flow across quick task, task edit, kanban card, and task detail dialog
</objective>
