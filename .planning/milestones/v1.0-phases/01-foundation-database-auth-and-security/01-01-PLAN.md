## Key Decisions (from CONTEXT.md)
- All tables have `created_at` and `updated_at` timestamps
- UUIDs for primary keys
- Hard delete (no soft deletes)
- Database-level foreign key constraints
- Team members CAN view client details for assigned tasks
- Team members CANNOT view other team members' profiles
- Team members CAN view comments and notifications on assigned tasks
- Storage: 10MB max, images only (.png, .jpg, .jpeg, .gif, .webp)
- Storage path: `{project_id}/{task_id}/{filename}`
- Invite tokens: 7-day expiry, 32-char random, generated on admin click
</context>

<task type="auto">
  <name>Task 2: Create RLS policies for all tables</name>
  <files>supabase/migrations/002_rls_policies.sql</files>
  <action>
    Create `supabase/migrations/002_rls_policies.sql` with RLS policies.

    First, enable RLS on all 8 tables:
    ```sql
    ALTER TABLE public.clients ENABLE ROW LEVEL SECURITY;
    -- (repeat for all 8 tables)
    ```

    Then create policies following these access rules:

    **Admin access** (checks `auth.jwt() -> 'app_metadata' ->> 'role' = 'admin'`):
    - Full CRUD (SELECT, INSERT, UPDATE, DELETE) on ALL 8 tables
    - One policy per table: `CREATE POLICY admin_all ON public.{table} FOR ALL USING ((SELECT auth.jwt() -> 'app_metadata' ->> 'role') = 'admin') WITH CHECK ((SELECT auth.jwt() -> 'app_metadata' ->> 'role') = 'admin');`

    **Team member access** (checks `auth.jwt() -> 'app_metadata' ->> 'role' = 'team_member'`):
    - clients: SELECT only, WHERE client has projects with tasks assigned to this member (join through projects → tasks → task_assignments WHERE team_member_id = auth.uid())
    - projects: SELECT only, WHERE project has tasks assigned to this member
    - tasks: SELECT + UPDATE, WHERE task has assignment to auth.uid() (UPDATE limited — can change caption, design_file_path, status)
    - task_assignments: SELECT only, WHERE team_member_id = auth.uid()
    - comments: SELECT only on assigned tasks, INSERT on assigned tasks
    - team_members: SELECT only own row (id = auth.uid())
    - invite_tokens: no access (admin-only)
    - notifications: SELECT + UPDATE (mark read), WHERE team_member_id = auth.uid()

    **No client-side RLS** for portal access (portal uses service role from server component — slug IS authorization).

    Use `(select auth.uid())` wrapper for statement-level caching per Supabase best practices.
  </action>
