  <action>
    Add a `linkify` function to `lib/utils.ts` that converts URLs in plain text to clickable anchor tags:

    ```typescript
    const URL_REGEX = /(https?:\/\/[^\s<]+)/g

    export function linkify(text: string): string {
      return text.replace(
        URL_REGEX,
        '<a href="$1" target="_blank" rel="noopener noreferrer" class="underline">$1</a>'
      )
    }
    ```

    The function uses a regex to match URLs starting with `http://` or `https://` and wraps them in `<a>` tags with `target="_blank"`, `rel="noopener noreferrer"`, and Tailwind `underline` class.

<task type="auto">
  <name>Task 2: Create task detail page with full edit form</name>
  <files>app/admin/clients/[clientId]/projects/[projectId]/tasks/[taskId]/page.tsx</files>
  <action>
    Create `app/admin/clients/[clientId]/projects/[projectId]/tasks/[taskId]/page.tsx`:

    This is a `'use client'` component because it needs interactive form editing. The data is fetched server-side and passed as props, OR it uses a server component wrapper that fetches data and renders an inner client component.

    Approach: Parent RSC fetches data, passes to client form component.

    **Server component** (default export):
    1. Accept params: `{ params: Promise<{ clientId: string; projectId: string; taskId: string }> }`.
    2. Import `createClient` from `@/lib/supabase/server`.
    3. Fetch the task by ID:
       ```typescript
       const { data: task } = await supabase.from('tasks').select('*').eq('id', taskId).single()
       ```
    4. Fetch all team members for the assignment dropdown:
       ```typescript
       const { data: teamMembers } = await supabase.from('team_members').select('id, name, email')
       ```
    5. Fetch current assignment (if any):
       ```typescript
       const { data: assignment } = await supabase
         .from('task_assignments')
         .select('team_member_id, team_members(name, email)')
         .eq('task_id', taskId)
         .single()
       ```
    6. Fetch project and client for breadcrumb.
    7. Pass all data to `<TaskEditForm>` client component.

    **Client component (`<TaskEditForm>`):**
    1. Use `useForm` from react-hook-form with `zodResolver(zodSchema)`.
    2. Zod schema matches task creation schema: title (required), briefing (optional), caption (optional), postingDate, dueDate, deadline, status (enum: todo, in_progress, done).
    3. Pre-populate form with `defaultValues: { title: task.title, briefing: task.briefing || '', caption: task.caption || '', ... }`.
    4. **Briefing field:** Below the briefing textarea, render a preview section showing the linkified briefing text using the `linkify()` function:
       ```tsx
       const BriefingPreview = ({ text }: { text: string }) => {
         if (!text) return null
         return <div dangerouslySetInnerHTML={{ __html: linkify(text) }} className="text-sm p-3 border border-border rounded-sm" />
       }
       ```
    5. **Team assignment dropdown:** `<select>` or `<Select>` (shadcn) with all team members. Current assignment pre-selected. On change: calls `assignTaskToMemberAction(taskId, memberId)` server action. Include a "Unassign" option (null member ID).
    6. **File upload section (D-12):** If task has `design_file_path`:
       - Show: "Current file: {filename}" with a download link using `<DesignFileDownloader path={task.design_file_path} fileName={filename} />`
       - Show a "Replace" button that triggers `<DesignFileUploader projectId={projectId} taskId={taskId} onUploadComplete={(path) => setDesignFilePath(path)} />`
       - After upload, call `updateTaskFilePathAction(taskId, newPath)` to persist the new path.
       If task has NO design file:
       - Show `<DesignFileUploader>` with "Upload design file" label.
    7. **Save button:** Calls `updateTaskAction(taskId, formData)` → on success, toast "Task saved" → `revalidatePath` on parent.
    8. **Delete button:** Red button, calls `deleteTaskAction(taskId, projectId, task.design_file_path)` → redirects to project tasks page.
  </action>
