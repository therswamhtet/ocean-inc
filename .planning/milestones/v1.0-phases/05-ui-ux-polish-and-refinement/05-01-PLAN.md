# Plan 05-01: Shared Component Audit — Modal Sizing, Terminology, Share Links

**Wave:** 1 (foundation)
**Phase:** 05-ui-ux-polish-and-refinement
**Date:** 2026-04-04
**Estimated Duration:** 8 min

## Goal

Audit and fix shared components across all surfaces: modal sizing, terminology consistency, and share link behavior. Establish `lib/labels.ts` as the single source of truth for UI strings.

## Tasks

### 05-01-01: Modal sizing audit and fix

**Audit** all Dialog component usages across the codebase:
- Admin: task creation/edit dialogs, invite modals, confirmation dialogs
- Team: notification dialogs, confirm dialogs
- Portal: task detail modal

**Fix** each Dialog to:
- Use responsive maxWidth: `max-w-[95vw]` on mobile → `max-w-xl` / `max-w-2xl` on desktop
- Add `max-h-[85vh] overflow-y-auto` to content containers for scrollable content
- Keep actions (buttons) visible outside scrollable area
- Test at 375px width

### 05-01-02: Terminology audit and centralized labels

**Audit** all UI labels across admin, team, and portal surfaces:
- Grep for label variants: "Post Date" vs "Posting Date" vs "Publish Date", "assigner" vs "admin", "project" vs "client project"
- Map all inconsistent variants to canonical forms

**Create** `lib/labels.ts` with centralized constants:

**Update** all surfaces to import from `lib/labels.ts`:
- Admin surface labels

### 05-01-03: Share link fixes

**Audit** client portal share link behavior:
- Verify absolute URL generation (include origin)
- Check CopyButton integration in admin area where admins can copy client portal link
- Verify "Copied!" feedback with timeout
- Check share link is accessible from client list view or client detail view

**Fix** any issues found:
- Ensure URL is absolute: `https://origin/portal/[slug]` or use `window.location.origin`
- Add or fix share link copy button in appropriate admin location
- Verify the link works when pasted in a new browser tab

## Verification

- [ ] All modals render correctly at 375px, 768px, and 1024px breakpoints
- [ ] `lib/labels.ts` exists and is imported across all surfaces
- [ ] No duplicate/conflicting labels remain (`rg "Post Date|Publish Date" "*.tsx"` returns 0 hits for old variants)
- [ ] Share link copies correctly and opens portal when pasted
- [ ] `npm run build` passes without errors

## Dependencies

- None (builds on existing component implementations from prior phases)

## Notes

- Only change visible UI text labels, NOT variable names, route paths, or data attributes
- Modal sizing should be done once and applied consistently, not piecemeal
- Share link should use existing CopyButton component pattern
