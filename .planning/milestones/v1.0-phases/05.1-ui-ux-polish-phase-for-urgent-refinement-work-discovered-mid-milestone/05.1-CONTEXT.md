# Phase 05.1: UI/UX Polish Phase for Urgent Refinement Work Discovered Mid-Milestone

**Gathered:** 2026-04-05
**Status:** Ready for planning
**Source:** User-provided requirements via /gsd-plan-phase 05.1

<domain>
## Phase Boundary

Phase 05.1 is an urgent UI/UX refinement pass addressing 12 specific issues discovered during Phase 5 execution and user feedback. This phase touches multiple surfaces: admin client management, task creation, dashboard, and shared components. No new database tables or API endpoints are required — this is purely a presentation and UX layer refinement.

</domain>

<decisions>
## Implementation Decisions

### Client Interaction
- **D-01:** Client card/island click target expanded — clicking anywhere on the client card (not just the name) opens the client detail modal
- **D-02:** Client cards must show share link button and calendar link for portal access

### Share/Invite Functionality  
- **D-03:** Generate Invite Link function must be fixed — currently non-functional
- **D-04:** Share links must work correctly for client portal access

### Task Creation Modal
- **D-05:** Task creation modal needs better responsive sizing — currently takes up too much screen space
- **D-06:** Task creation modal must include assignee field
- **D-07:** Users must be able to assign tasks to themselves from the task creation modal

### Terminology
- **D-08:** All instances of "admin console" terminology must be changed to "management panel" throughout the app

### Notifications
- **D-09:** Dashboard notification redundancy — remove duplicate "notification" text label, keep only bell icon with badge, or merge into unified notification section

### Mobile Navigation
- **D-10:** Hamburger menu must move from left side to right side on mobile view (<768px)

### Dashboard Enhancements
- **D-11:** Admin dashboard must include a small calendar view showing tasks due on each date
- **D-12:** Admin dashboard must include a dedicated task view with sections for: Upcoming Tasks, Today's Tasks, Overdue Tasks (dashboard-style layout, not just client-project drill-in)

### Client Card Redesign
- **D-13:** Client cards need visual polish — add color options and improve visual appeal (while maintaining B&W brand constraints)

### Button Polish
- **D-14:** Task/project creation modal buttons must have proper sizing — no overly long stretched buttons

</decisions>

<specifics>
## Specific Ideas

### Client Card Improvements
- Expand click target to entire card area
- Add color/accent options within B&W constraints (subtle gray variations)
- Ensure share link and calendar links are visible

### Task Modal Polish
- Responsive max-width: sm:max-w-lg for desktop, full-width on mobile
- Include assignee dropdown that defaults to self
- Proper button sizing with min-w-[120px] or auto-width

### Mobile Hamburger Repositioning
- Move from `left-0` to `right-0` on mobile header
- Adjust SheetContent `side="left"` to `side="right"`

### Dashboard Calendar
- Small monthly calendar grid on dashboard
- Shows dots on dates with tasks
- Click date to see tasks due that day

### Dashboard Task View
- Three sections: Overdue, Today, Upcoming
- Cards with task title, client, project, due date
- Status dot indicator

### Notification Cleanup
- Remove redundant "Notifications" text label near bell
- Keep bell icon with unread badge only

### Invite Link Fix
- Debug current implementation
- Ensure it generates valid invite token
- Show appropriate feedback on success/failure

</specifics>
