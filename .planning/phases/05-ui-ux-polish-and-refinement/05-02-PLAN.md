# Plan 05-02: Mobile Navigation + Responsive Tables

**Wave:** 2 (surface-level polish)
**Phase:** 05-ui-ux-polish-and-refinement
**Date:** 2026-04-04
**Estimated Duration:** 10 min

## Goal

Implement mobile navigation for admin and team areas, make client portal tabs responsive on small screens, and convert all table views to card layouts on mobile.

## Tasks

### 05-02-01: Admin sidebar mobile navigation

**Implement** hamburger menu for admin sidebar on screens <768px:
- Sidebar collapses to hamburger icon (top-left) at `<md` breakpoint
- Menu slides in from left with overlay backdrop
- Same links, same grouping, just different layout
- Close button inside slide-out panel
- Overlay click closes menu
- Hamburger icon: lucide-react `Menu` and `X` icons

**Preserve** desktop layout:
- All `md:` and larger breakpoints keep existing wide sidebar
- No changes to desktop sidebar styling

### 05-02-02: Team sidebar mobile navigation

**Mirror** admin mobile nav pattern for team sidebar:
- Same hamburger toggle, same slide-out panel
- Team-specific nav items preserved
- Consistent animation and styling with admin nav

### 05-02-03: Client portal tabs responsive

**Update** portal tab navigation (Kanban | Calendar | Timeline) for mobile:
- Horizontal scroll on very narrow screens if tabs overflow
- OR stack tabs vertically below the page title at <500px
- Active tab indicator works in both horizontal and stacked modes
- Touch targets minimum 44px height

### 05-02-04: Table-to-card conversion for mobile

**Convert** all table views to card layouts on mobile (<768px):
- Admin client list (table rows → stacked cards with client name + project count)
- Admin project list under client → stacked cards with project name, month, status
- Admin task list → stacked cards with title, assignee, due date, status dot
- Team task list → same card pattern as admin but team-focused fields
- Each card shows the same data as its table row, just reformatted
- Hide table header on mobile, show cards instead

## Verification

- [ ] Admin sidebar collapses to hamburger at <768px, full sidebar at ≥768px
- [ ] Team sidebar mirrors admin pattern
- [ ] Client portal tabs stack or scroll on small screens
- [ ] All table views render as card lists at <768px
- [ ] All cards show same info as table rows (no data loss)
- [ ] Touch targets ≥44px on all mobile interactive elements
- [ ] `npm run build` passes without errors

- Admin and team layout files from Phase 2 and Phase 3

- Consider a reusable `MobileCard` wrapper for card-layout-on-mobile pattern
- Hamburger menu state is client-side (useState in layout component)
