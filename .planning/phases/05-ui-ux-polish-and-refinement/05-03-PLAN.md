# Plan 05-03: Calendar Polish + Dashboard Redesign

**Wave:** 2 (surface-level polish)
**Phase:** 05-ui-ux-polish-and-refinement
**Date:** 2026-04-04
**Estimated Duration:** 10 min

## Goal

Polish the calendar view for mobile usability, redesign the admin and team task dashboards with improved visual hierarchy, and make notifications section collapsible.

## Tasks

### 05-03-01: Calendar view polish

**Update** portal calendar component (from Phase 4):
- Minimum cell height of 44px for touch targets
- Horizontal scroll wrapper for narrow viewports that can't fit all days
- Task indicators truncate with ellipsis for multi-task days
- Week/month toggle button has clear visual state and 44px minimum size
- Calendar grid maintains B&W brand (no colored backgrounds, only borders)
- Previous/next month navigation buttons have 44px minimum touch targets

**Update** admin calendar (if exists):
- Same polish as portal calendar for consistency
- Add any interactive elements present in admin but not portal

### 05-03-02: Admin dashboard redesign

**Redesign** `app/admin/page.tsx` dashboard:
- 4 metric cards in responsive grid: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4`
- Metrics: Total Active Projects, Tasks In Progress, Overdue Tasks, Completed This Month
- Large number display (`text-3xl`), smaller label (`text-sm text-[#888888]`)
- Consistent card styling (border, padding) matching new card tokens
- Notifications section below metrics, collapsible with toggle
- Collapsed state shows count badge only, expanded shows full list

### 05-03-03: Team dashboard redesign

**Redesign** team dashboard (`app/team/page.tsx`):
- Same 4-metric responsive grid pattern as admin
- Metrics: Total Assigned, Due Today, Overdue, Completed
- Consistent card styling with admin dashboard
- Task list below summary cards uses the same card layout from 05-02 task conversion
