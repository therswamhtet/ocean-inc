# Plan 05-04: Card Redesign Unification

**Wave:** 3 (consistency pass)
**Phase:** 05-ui-ux-polish-and-refinement
**Date:** 2026-04-05
**Estimated Duration:** 8 min

## Goal

Audit all card-like components across admin, team, and portal surfaces. Unify shared tokens (border, radius, padding, spacing) so every card variant — kanban, list, metric, mobile card — follows identical styling tokens. Ensure no card diverges from the established pattern.

## Tasks

### 05-04-01: Audit card components across all surfaces

**Review** every card component:
- `components/admin/kanban-card.tsx` — admin kanban cards
- Portal kanban cards (in portal views)
- Dashboard metric cards (admin `app/admin/page.tsx`, team `app/team/page.tsx`)
- Mobile card layouts (in `components/admin/task-list.tsx` and mobile sections of client/project pages)
- Any other card-like containers

**Document** any divergence from the shared tokens:
- Border: should be `border border-border`
- Radius: should be `rounded-lg` or `rounded-sm` (max 8px per brand)
- Padding: should be `p-3` or `p-4` consistently
- Hover: should be `transition hover:bg-muted/30`
- Spacing between cards: consistent gap values

### 05-04-02: Create shared card wrapper component

**Create** `/components/ui/content-card.tsx` — a lightweight reusable card wrapper:

```tsx
import { cn } from '@/lib/utils'
import React from 'react'

interface ContentCardProps extends React.HTMLAttributes<HTMLDivElement> {
  variant?: 'default' | 'metric' | 'kanban' | 'mobile'
}

export function ContentCard({ className, variant = 'default', ...props }: ContentCardProps) {
  const base = 'rounded-lg border border-border transition'
  const variants = {
    default: 'p-4 hover:bg-muted/30',
    metric: 'p-5',
    kanban: 'p-3 hover:bg-muted/30',
    mobile: 'p-3',
  }

  return <div className={cn(base, variants[variant], className)} {...props} />
}
```

**Replace** inline card classNames with this component across:
- Admin kanban cards
- Dashboard metric cards
- Mobile card layouts
- Any remaining inline card patterns

### 05-04-03: Verify client island interactivity

**Update** client list so clicking anywhere on the client's card/island opens the project modal — not just clicking the name. Ensure the entire card is an interactive surface area.

**Verify** the click target is the full card, not just inner text elements.
