<objective>
Implement inline task editing on the Kanban board without navigation, expand dropdown hit area on task list rows, and add posting_time support to the database.
</objective>

<research_context>
Phase 9 has 5 success criteria:
1. Admin can click a Kanban card and edit fields inline without navigating away
2. Task dropdown menus open when clicking any area of the task row
3. Task details dialog redesigned
4. Image previewer loads correctly with preview sizes
5. Admin can set posting time with default 10:00 AM

This plan covers: inline Kanban editing (TASK-01, TASK-02), task list dropdown expand (TASK-03), and database schema for posting_time (TASK-06).
</research_context>

<action>
Create an inline edit dialog for Kanban cards. In components/admin/kanban-card.tsx:

1. Remove the `<Link>` that wraps the title
2. Add an `<Dialog>` component that opens on card click (not navigation)
3. Create inline edit form with fields: title, posting_date, status, assigned_to
4. Use existing task-edit-form.tsx as reference for the form fields
5. Pass task ID and projectId to the dialog
6. On save, call updateTaskStatusAction and also create updateTaskDetailsAction for title/posting_date changes

</action>

Add posting_time column to the tasks table:

1. Create database migration: `ALTER TABLE tasks ADD COLUMN posting_time TIME DEFAULT '10:00:00'`
2. The column should be nullable (TIME) with default '10:00:00'
3. In actions.ts, add updateTaskTimeAction that updates posting_time for a task
4. Update TaskRow type in task-view-toggle.tsx to include: `posting_time: string | null`
5. Update KanbanCard to display posting_time alongside posting_date when set
6. Format: "10:00 AM" display using date-fns format function

The default time of 10:00 AM applies when no specific time is set.
</action>
