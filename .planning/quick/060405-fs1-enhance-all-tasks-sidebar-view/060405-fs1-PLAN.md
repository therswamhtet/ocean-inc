# Quick Task Plan: 060405-fs1

**Task:** Enhance All Tasks sidebar view with full task details and expandable detail panel

**Date:** 2026-04-06

## Goal

Enhance the All Tasks view in the admin sidebar (Tasks tab) to display richer task information and provide an expandable detail panel when clicking on a task row.

## Changes

### 1. New `AllTasks` client component (`app/admin/tasks/all-tasks.tsx`)
- Expandable task rows with chevron toggle
- Rich inline view showing: status dot, task title (linked to parent), content plan/month, client name with color dot, project name, due date, assignee
- Expandable detail panel showing: full task info, assigned to, posting date, content plan, due date, deadline, and "Open task detail" link

### 2. Updated `app/admin/tasks/page.tsx`
- Rewrote `fetchTasks` to join `projects` and `clients` (avoiding `!inner` joins that trigger RLS recursion)
- Added `task_assignments` left join for assignee name
- Returns flattened data with client_name, client_color, project_name, assigned_to_name

### 3. Fixed null-safety bugs in `dashboard-inner.tsx`
- `DashboardCalendar`: Added `(tasks ?? [])` to prevent reduce on null
- `DashboardTaskSections`: Added `(overdueTasks ?? [])` and `(todayTasks ?? [])` null coalescing

### 4. Fixed RLS recursion in `clients/page.tsx`
- Removed `projects` join from clients query to avoid RLS policy recursion
- Used separate `projects` query to get active project counts per client

### 5. Added `createServiceRoleClient` to bypass RLS recursion (`lib/supabase/server.ts`)
- Service role client bypasses RLS entirely, avoiding the recursion issue
- Used for ALL queries that could trigger the recursive EXISTS subquery chain

### 6. Updated pages to use service role client for ALL queries involving clients/projects/tasks
- `app/admin/tasks/page.tsx`: All queries use service role client
- `app/admin/clients/page.tsx`: All queries use service role client
- `app/admin/page.tsx`: All queries (including count queries) use service role client
- `app/admin/clients/[clientId]/page.tsx`: Client and projects queries use service role client
